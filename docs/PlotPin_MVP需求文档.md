# PlotPin —— 实验数据图表工具 MVP 需求文档

## 1. 项目目标
**PlotPin** 是一个个人用的网页小工具:上传一个 CSV 文件 → 配置坐标轴和图表标题 → 生成折线图 → 得到一个**唯一且永久不变的链接**,可插入到笔记中引用。单人使用,部署在绿联 NAS 上。

## 2. 部署环境
- 运行在绿联 NAS(UGOS Pro 系统,已支持 Docker / docker-compose)
- 打包为 Docker 容器,用 `docker-compose` 启动
- 单人使用,**无需登录 / 账号系统**
- 局域网访问(同网络下电脑、手机、平板都能打开链接)
- **数据必须持久化**:CSV 文件和数据库通过 Docker volume 映射到 NAS 的共享文件夹,容器重建 / 升级后数据不丢失

## 3. 核心用户流程
1. 打开网页,上传一个 CSV 文件
   - 格式约定:**第一列 = 横坐标(X)**,**第二列及以后 = 纵坐标(Y,可多列)**
   - CSV **含表头行**,表头作为每条曲线的名称(图例)
2. 弹出配置窗口,用户输入三个标题:
   - X 轴标题
   - Y 轴标题
   - 图表标题
3. 生成并展示折线图:
   - 第二列起的每个 Y 列画成一条曲线,**默认全部画在同一幅图上**
   - 用**颜色 + 图例**区分各曲线,图例名取自表头列名
4. 生成一个**唯一、永久不变的链接**,用户可拷贝插入笔记

## 4. 链接 / 输出
每张图分配一个唯一 ID,提供两个永久地址(ID 不变 → 链接永久不变):
- **交互式页面**:`http://<NAS地址>/chart/<id>` —— 浏览器打开,可悬停看数值、缩放
- **静态图片**:`http://<NAS地址>/chart/<id>.png`(建议同时支持 `.svg`)—— 可用 `![](url)` 形式直接嵌入笔记中显示

两个地址共用同一个 ID。

## 5. 数据存储
每个图表记录需持久保存,以保证链接永久有效:
- 原始 CSV 文件(存文件系统)
- 配置:X 轴标题、Y 轴标题、图表标题
- 唯一 ID、创建时间

建议:配置 / 元数据用 SQLite,CSV 存文件系统;两者都放在持久化卷里。

## 6. 技术栈建议(供参考,可由 Claude Code 调整)
- 后端:Python + FastAPI
- CSV 解析:pandas
- 交互式图表:前端用 Plotly.js(自带悬停 / 缩放)
- 静态图片:后端用 matplotlib 渲染 PNG / SVG(稳定、无需无头浏览器,适合 NAS 容器环境)
  - 说明:交互页面与静态图片是两条渲染路径,样式可能略有差异,MVP 阶段可接受
- 数据库:SQLite
- 打包:`Dockerfile` + `docker-compose.yml`(容器 / compose 服务命名为 `plotpin`)

## 7. 明确不做(MVP 范围外,避免过度开发)
以下均为后续迭代内容,**本次不要实现**:
- 分组、标签、检索
- 仪器配置(profile)、自动识别格式
- `.mat` 文件支持
- 折线以外的其他图型、对数坐标、手动选列
- 大文件分块续传、收件夹登记(MVP 假设 CSV 不大,普通上传即可)
- 编辑已生成的图表、用户账号系统

## 8. 实现注意点
- 服务端 / 反向代理如有上传体积限制,适当放宽(即便 MVP 文件不大,留点余量)
- 容器数据卷必须映射到 NAS 实际目录,确保持久化
- CSV 编码:优先按 UTF-8 读取,建议对常见编码(如 GBK)做兼容兜底
- 链接 ID 用不可猜测的短随机串(如 8–12 位),避免顺序可枚举
