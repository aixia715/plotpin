# PlotPin 设计文档

> 基于 `docs/PlotPin_MVP需求文档.md`,经头脑风暴细化后的设计方案。
> 日期:2026-06-21

## 1. 目标

PlotPin 是单人自用的网页小工具:上传 CSV → 配置坐标轴与标题 → 生成折线图 → 得到唯一且永久不变的链接,可嵌入笔记引用。部署在绿联 NAS,Docker 容器运行,局域网访问,无登录系统,数据持久化。

## 2. 技术栈

- 后端:Python + FastAPI + Jinja2 模板(方案 A:后端直出静态 HTML)
- CSV 解析:pandas + 自写工程计数法解析模块
- 交互图:Plotly.js,**本地打包进静态资源**(不走 CDN——局域网/NAS 环境,看笔记的设备可能无外网)
- 静态图:matplotlib,使用 `EngFormatter` 渲染工程计数法刻度,输出 PNG / SVG
- 数据库:SQLite
- 打包:Dockerfile + docker-compose.yml,服务名 `plotpin`

## 3. 目录结构

```
plotpin/
├─ app/
│  ├─ main.py          # FastAPI 路由
│  ├─ storage.py       # SQLite + 文件系统读写

│  ├─ eng_notation.py  # 工程计数法 解析/格式化(解析与渲染共用)

├─ tests/
├─ Dockerfile
├─ docker-compose.yml

```

## 4. 工程计数法约定(核心)

CSV 中所有数值单元格(X 列与 Y 列)均采用工程计数法书写:

- 尾数(数值部分)的**幅值**须落在区间 `[1, 1000)`(含 1、不含 1000),或为 `0`;超出范围须用 SI 词头表达数量级。例:`1`、`1k`(=1000)、`999.9` 合法;`0.5`、`1000` 非法(应写 `500m`、`1k`)。
- **支持负号**:允许 `-3.3`、`-47k`,符号之外的幅值仍须满足上述区间约定。
- SI 词头(**大小写敏感**)与倍率:

  | 词头 | f | p | n | u | m | (无) | k | M | G | T |
  |---|---|---|---|---|---|---|---|---|---|---|
  | 倍率 | 1e-15 | 1e-12 | 1e-9 | 1e-6 | 1e-3 | 1 | 1e3 | 1e6 | 1e9 | 1e12 |

  注意:`m`=毫 与 `M`=兆,`u`=微 与 `M`=兆,均区分大小写。

- 示例:`47k`=47000,`100m`=0.1,`3.3`=3.3,`0`=0。

**解析与显示分离**:
- 解析:CSV 数据始终被还原成浮点数用于绘图(固定行为)。
- 显示:配置项 `x_eng` / `y_eng` 仅控制坐标轴刻度与悬停数值的**显示风格**。勾选则以工程计数法显示(如 `47k`),取消则显示普通数字(如 `47000`)。默认两者都勾选。
- 交互图(Plotly,手动格式化刻度与 hover)与静态图(matplotlib `EngFormatter`)共用同一份 `eng_notation` 格式化逻辑,保证两条渲染路径风格一致。

## 4b. 对数坐标(每轴可选)

- 配置项 `x_log` / `y_log` 各自独立控制对应轴是否为**对数坐标**(默认不勾选,即线性)。
- 与工程计数法显示**正交**:工程计数法只管刻度文字怎么写,对数只管轴的刻度间距怎么排;四个开关(`x_eng`/`x_log`/`y_eng`/`y_log`)互不影响。对数轴的刻度文字仍按对应轴的 `*_eng` 决定用工程计数法还是普通数字。
- **硬约束**:对数对 `≤0` 无定义。若某轴勾选对数、但该轴数据含 `0` 或负数,**严格拒绝**(见 §7),不静默丢点。
- 实现:交互图 Plotly 用 `xaxis.type/yaxis.type = "log"`;静态图 matplotlib 用 `set_xscale/set_yscale("log")`。对数轴刻度用十进制(decade,即 10 的整数幂)位置,文字仍走 `eng_notation` 格式化。

## 5. 数据模型

SQLite 表 `charts`:

| 字段 | 类型 | 说明 |
|---|---|---|
| id | TEXT 主键 | 8–12 位不可猜测短随机串(避免顺序枚举) |
| title | TEXT | 图表标题 |
| x_title | TEXT | X 轴标题 |
| y_title | TEXT | Y 轴标题 |
| x_eng | INTEGER | X 轴是否工程计数法显示 (0/1) |
| y_eng | INTEGER | Y 轴是否工程计数法显示 (0/1) |
| x_log | INTEGER | X 轴是否对数坐标 (0/1) |
| y_log | INTEGER | Y 轴是否对数坐标 (0/1) |
| created_at | TEXT | 创建时间 (ISO 8601) |

- CSV 原文件按 `data/csv/<id>.csv` 存盘(不进数据库)。
- 渲染缓存按 `data/cache/<id>.png` 与 `data/cache/<id>.svg` 存盘。
- 曲线图例名直接取自 CSV 表头(第二列起每列一条曲线),不单独存储。
- `data/` 整个目录映射为 Docker 卷到 NAS 共享目录,确保容器重建/升级数据不丢。

## 6. 数据流

### 创建(一次提交,无临时文件)

1. 首页选文件 → 前端弹出配置窗口(纯客户端 UI),填三标题 + 每轴两个勾选(工程计数法显示、对数坐标)。
2. 提交 → `POST /charts`(multipart:文件 + 配置字段)。
3. 后端处理:
   - 识别编码:优先 UTF-8,失败兜底 GBK。
   - pandas 读取(第一列=X,第二列起=Y,含表头行)。
   - 逐格按工程计数法解析校验。
   - 对数校验:若 `x_log` 且 X 数据含 `≤0`,或 `y_log` 且任一 Y 值含 `≤0` → 拒绝。
   - 任一格非法/空值、对数与 ≤0 冲突,或文件级错误 → **不创建记录**,回到上传页显示中文错误。
   - 全部通过 → 生成 ID → 存 CSV → 写数据库 → 302 跳转 `/chart/<id>`。

### 查看

- `GET /` → 上传区 + 图表清单(按 created_at 倒序,显示标题 + 创建时间,点击进入对应图表页)。MVP 不做检索/分组;编辑、删除入口留待后续迭代。
  - **缩略图(方案一)**:每条记录左侧显示缩略图——**仅当该图表的 `.png` 缓存文件已存在时**才用它(`<img>` 缩小展示),否则显示占位图标。**首页绝不为缩略图触发渲染**,以免大文件拖慢首页加载。即首页只读 `cache_path(id, "png").exists()`,不调用 `render_static`。
- `GET /chart/{id}` → 渲染交互页;解析后的数值数据以 JSON 内嵌进页面,Plotly.js 绘图(默认所有 Y 列画在同一幅图,颜色+图例区分)。页面下方提供可一键拷贝的链接:交互页链接、`.png` 链接、`.svg` 链接。
- `GET /chart/{id}.png` / `GET /chart/{id}.svg` → 命中缓存直接返回;否则 matplotlib 现渲染 → 写 `data/cache/` → 返回(按需生成 + 永久缓存,因 MVP 不支持编辑,缓存永久有效)。
- 访问不存在的 id → 404 页。

## 7. 错误处理

| 场景 | 处理 |
|---|---|
| 非 CSV / 无法解析为表格 | 上传页提示「文件无法识别为 CSV」 |
| 编码识别失败(UTF-8、GBK 均失败) | 提示「无法识别文件编码」 |
| 列数不足(仅 1 列,无 Y 列) | 提示「至少需要 X 列和 1 个 Y 列」 |
| 表头缺失/重复 | 提示具体问题 |
| 单元格非法(非法词头、数值部分超 1–1000 范围、空值/缺失) | 严格拒绝,提示「第 N 行 第 M 列 `值` 无法解析」 |
| 轴设为对数但该轴数据含 ≤0 | 严格拒绝,提示「X(或 Y)轴含 ≤0 值,无法使用对数坐标」 |
| 访问不存在的 id | 404 页面 |
| 上传体积限制 | 反向代理 / FastAPI 上传上限适当放宽,留余量 |

策略说明:实验数据对准确性敏感,采用**严格模式**——宁可报错也不画出看着对、其实有问题的图。空单元格/缺失值同样视为错误。

## 8. 测试策略(TDD,pytest)

- `eng_notation`:解析与格式化双向用例——
  - 解析:`47k→47000`、`100m→0.1`、`0→0`、`3.3→3.3`、`1k→1000`、`-47k→-47000`;非法 `47x` 报错;超范围 `4700`(无词头但 ≥1000)、`0.5`(<1 无词头)报错;大小写 `m` vs `M`、`u` vs `M`;边界 `1` 合法、`1000` 非法。
  - 格式化:`47000→47k` 等反向用例。
  - 这是核心模块,用例需密集覆盖。
- `parsing`:好 CSV(单/多 Y 列、UTF-8/GBK 编码)正确读取;各类坏 CSV 正确报错。
- `ids`:长度、字符集、唯一性。
- 路由集成测试(FastAPI TestClient):上传成功→跳转;上传坏文件→报错;`GET /chart/{id}`;`.png`/`.svg` 渲染与缓存命中。

## 9. MVP 范围外(本次不做)

分组/标签/检索、仪器 profile、自动识别格式、`.mat` 支持、折线以外图型、手动选列、大文件分块续传、编辑/删除已生成图表、用户账号系统。(首页图表清单本次做,但清单上的编辑/删除留待后续。对数坐标原列为范围外,现已纳入本版,见 §4b。)

**数据规模约定(本版)**:本版假设单个 CSV 的数据点在**几千个**量级,由用户自行控制输入规模;不做降采样,交互图直接内嵌全部数据点。**后续**:对大文件做**降采样显示**(关键约束:降采样必须**保留每段的极大值与极小值**,避免丢失峰谷),原始 CSV 始终完整保存、链接永久有效。
